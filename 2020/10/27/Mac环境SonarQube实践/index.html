阅读预警

本文讨论基于MacOS系统，Windows系统没有具体实践过。

SonarQube是什么

按照他的自我介绍： 代码质量和安全工具。

直白的说就是检查代码中警告、错误、语法问题、重复代码等的工具。SonarQube把这些问题分为三类： Bugs（有明显的问题）、异味（虽然看起来没问题，但不推荐这么写）、重复（CV工程师不好当啊）

SonarQube 将很多文字类的统计图形化，有利于大家去伪存真。有重点的展示结构也可以让大家关注点在代码质量提升上。

支持的语言

原生支持JAVA。也支持其他语言，例如C#、JS、Python、PHP、C++、Object-C、Swift等。

Tips： 原生支持就是配置简单的意思，也支持就是配置特别复杂的意思……

基本了解

SonarQube 操作环境分为环境配置和SonarQube的使用。作为团队工具，搭建通常只需要一个人搭建，然后放到公共IP上访问。大部分人需要了解的就是使用，使用不是直接可以使用的，也需要根据不同编程语言、不同操作系统进行运行环境搭建。

SonarQube使用

环境

若Mac要使用SonarQube检查OC或Swift的代码质量，需要环境配置。

具体需要：

  • Java环境
  • 支持插件
  • OCLint
  • xcpretty

Java 环境

首先在终端输入命令java -version 查看Java版本。没有代表Java 未安装，需要重新安装JDK。

Mac 默认带JDK 1.8。若安装失败或者其他原因，到Java 官网下载最新的JDK8。

支持插件

下载Swift插件，放到SonarQube启动服务器的SonarQube/extensions/plugins/目录下。

OCLint

最关键！

检查是否完成oclint -version，若有OClint信息即为安装成功。

若不成功，比较麻烦。下面只讲解大致安装过程。

首先下载最新源码。还需要安装CMake,然后安装Ninja然后通过OCLint编码编译，Ninja快速编译器的依赖re2c也需要安装，然后进入源码的oclint-scripts，执行./make获得可执行文件(Release目录下)。

然后将可执行文件拉到/usr/local/下,编辑.bash_profile,加入:

进入目录中加上权限sudo chmod -R 777 ./，并且执行source ~/.bash_profile 使之生效。

xcpretty

gem install xcpretty

Demo 初体验

下载SonarQube iOSDemo。简单了解下是如何使用和运行的。

在Demo中 SonarDemo/目录下最重要的文件：

  • sonar-project.properties 配置文件
  • run-sonar-swift.sh 运行脚本

sonar-project.properties 需要修改很多东西，而运行脚本不需要修改。

打开Sornarqube 网页端图形化操作，例如打开http://10.0.1.44:9000/projects。点击右上角+号新增项目，创建名称或者标识，然后创建令牌。点击继续，选择其他语言，MacOS系统。按照要求下载sonar-scanner扫描器。将扫描器移动到/usr/local下，进入目录中并且加权限sudo chmod -R 777 ./。
